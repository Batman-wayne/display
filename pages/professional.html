Professional

Professional workplace experience I have had, along with what skills I learned.

Kratos | RT Logic
Summer 2020

Due to COVID-19, my internship at Kratos was online. I had two projects. The first project was a Python migration task. The company has a python script which intercepts HTTP requests from a website, grabs data from a map server, and then sends this data back to the webpage. The existing python code was written in python 2.7 and the original requirement was to rewrite this script in python 3. A coworker actually finished this project using C# before I was able to finish it using Python, but it was still a valuable learning experience.


My second project was writing a thread safe queue that could receive RF packets and then parse them. One thread would continuously receive packets and place them on a queue, and another thread pops the packets and parses them. This project was extremely interesting and allowed me to explore multithreading in a professional environment.

Kratos | RT Logic
Summer 2019

I went to the company in Colorado Springs, Colorado for the summer. This was really my first introduction to a defense constractor and I really enjoyed it. My summer was split into two tasks.


My first task was software. The team I was on worked on a tool that could take in satellite transponder data and output many useful visuals primarily in the field of Space Situational Awareness, such as satellite position prediction and bandwidth utilization. This tool's frontend was programmed using Angular JS, and my first task was to create a testing suite for the UI. Once I finished that, I moved on to add some features and fix some bugs with the GUI.


My second task was hardware. There was a narrow-bandwidth high-speed FPGA-based digitizer, which was designed to sit at the base of an antenna and in real time, quantize incoming packets. The digitizer had actually been put aside, with no progress made on it for a year due to the fact that the clock on the board was not locking. After quite a lot of hours spent in the lab, the error was revealed: the input clock signal was specified by the datasheet to be AC-coupled, but by accident was DC-coupled. Fixing this issue allowed the board to function properly and move on to next phases of testing.

Continental Automotive Summer 2018

I worked on a board which was intended to become an electronic torque redistribution tool for a car. The board needed to have a diagnostic mode, so that engineers working on the board could enter an "admin" mode and run any command. I added a new CAN header which would allow the user to do just that. I had worked with CAN bus previously with Illini Formula Electric, but this was my first working with CAN in a professional environment. Instead of using Arduinos with CAN shields, I got to use the Vector suite of CAN tools to test out my code and make sure it was working.


I also debugged a watchdog timer on the board, which was continuously sending a reset signal back to the microprocessor, rendering the board useless. After some debugging, it was attributed to an erroneous line of code triggering a reset.

Weber Packaging Solutions Summer 2017

This was my first internship. I worked on python code to automate a number generation process. I also learned how to use Visual Studio to develop windows apps using the .net framework and C#. I also used Inventor to update CAD drawings that had 1000+ parts.
