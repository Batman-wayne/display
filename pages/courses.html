Courses

Courses that I've taken that are relevant to my major and research areas. Arranged in reverse chronological order.

ECE 391: Operating Systems

This course is all about the synthesis between hardware and software, working together to make a Linux-like operating system. This starts from nothing more than a few boot files. The final operating system has a read-only filesystem, interrupt support, device support (Keyboard, Interrupt Controller, Real time clock), paging, and a round-robin scheduler for multitasking. The operating system was completed in a group of 4.


Concepts covered:

  • Interfacing C to assembly, macros, stack frame and calling convention
  • Synchronization: primitives, memory semantics, mutual exclusion, semaphores, scheduling, and race conditions
  • Interrupts: controlling generation and handling, chaining, cleanup code, interactions with device functionality
  • Resource management: virtualization and protection, virtual memory and hardware support
  • Exceptions and signals: exceptions due to instructions, memory accesses, and floating-point operations; signal semantics and delivery
  • Device programming: basic abstractions, character and block devices
  • File system abstractions: disk layout, access control lists and capabilities, log-structured and traditional filesystems as design problem
  • I/O interface: file descriptors, buffering, control operations, memory mapping

ECE 408: Applied Parallel Programming

This course teaches how to use CUDA C++ to drastically speed up the speed of sequential programs. I really liked this course, and ran some of the algorithms on my personal GPU.


Concepts covered:

  • CUDA basics
  • Tiling
  • Reduction and Scan operations
  • Tiling to speed up convolution
  • Different methods of sparse matrix multiplications
  • Speeding up raytracing using CUDA

ECE 448: Introduction to Artificial Intelligence

This course covers several topics relating to search, classification, reasoning, machine learning, games, planning, robotics, computer vision, and natural language understanding. Among the most relevant and useful of these was using PyTorch to quickly test and tune neural nets.


Concepts covered:

  • AI languages and formalisms
  • Problem solving
  • Knowledge representation
  • Deductive inference
  • Inductive inference and machine learning
  • Natural language understanding
  • Computer vision (Briefly)
  • PyTorch

ECE 310: Digital Signal Processing

This course covers the theory of Digital Signal Processing. The course was heavy in mathematics, but I've seen the concepts from that class show up in several places: the lab, my internships, etc. I also took the associated lab section, where we used Python to visualize a lot of the concepts taught in the classroom.


Concepts covered:

  • Discrete-time linear shift-invariant systems
  • Difference equations
  • Convolution and impulse response
  • z-transform
  • Transfer functions and block diagrams
  • Discrete-time Fourier transform (DTFT)
  • Sampling Theory
  • A/D and D/A conversion
  • FIR and IIR filter structures
  • Generalized linear phase
  • FIR and IIR filter design
  • Discrete Fourier transform (DFT)
  • ast Fourier transform (FFT)

ECE 374: Introduction to Algorithms and Models of Computing

This course covers many topics, most of which I was not a fan of. In fact, the only part of the course I really did like was near the end, where we discussed how to speed up algorithms and some pretty interesting graph algorithms.


Concepts covered:

  • Strings and languages
  • Finite automata (DFA, NFA) and regular expressions
  • Context free grammars and languages
  • Recursion, Divide and conquer, Backtracking
  • Dynamic programming
  • Greedy
  • Basic graph algorithms: reachability, BFS, DFS, shortest paths, MST, modeling via graphs
  • Polynomial-time reductions, P, NP, NP-Complete, NP-Hard

ECE 385: Digital Systems Laboratory (FPGA development)

This class covers several topics that are important to the field of programming and using FPGAs. Similar to parallel programming, impressive speedups can be achieved using FPGAs. This was one of my favorite courses.


Concepts covered:

  • Combinational logic circuits
  • Storage elements
  • Hazards and race conditions
  • Circuit characteristics (fanout, delays, etc.)
  • Field Programmable Gate Arrays (FPGAs)
  • Combinational networks (adders, multiplexes, etc.) in SystemVerilog
  • Sequential networks (counters, shift registers, etc.) in SystemVerilog
  • Synchronous state machines
  • Static timing analysis, clock domains, metastability, and synchronization
  • Logic simulation and testbenches
  • Microprocessors and system on chip

CS 498: Internet of Things

This course had several projects that introduced me to networking and general IoT applications. The most prominent and relevant of these was AWS.


Concepts covered:

  • IoT device programming (Arduino and Raspberry Pi)
  • IoT protocol stacks (Zigbee, 5G, NFC, MQTT, etc)
  • networking backhaul design and security enforcement
  • cloud-based IoT platforms such as AWS IoT

CS 225: Data Structures

This is one of the most important courses I've taken in my undergrad. Very useful skills on how to use data structures for interesting and unique problems.


Concepts covered:

  • C++ Programming
  • Linked Lists
  • Stacks and Queues
  • Trees
  • Dictionaries
  • Priority Queues
  • Disjoint Sets
  • Graphs
